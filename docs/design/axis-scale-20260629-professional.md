# 专业坐标轴系统设计

## 背景

当前 3D 坐标轴已经具备字段名、范围和简单刻度，但刻度按原始范围等分，容易出现 `-0.4486`、`0.1817` 这类不利于阅读的数值。同时轴线带有额外 padding，刻度和数据包围盒没有严格对齐，整体更像调试辅助，而不是专业绘图坐标系统。

## 设计目标

1. 坐标轴贴合数据包围盒，而不是悬浮在任意位置。
2. 主刻度使用 nice number，例如 `0.2`、`0.5`、`1`、`2`、`5`、`10k`。
3. 主刻度显示标签，次刻度只显示短线。
4. 底面和高度方向提供参考网格，帮助判断空间比例。
5. 坐标系统继续支持点云和曲面，不改变 core 中的数据归一化。

## 最小可交付方案

### AxisScale

每个轴从 `NumericStats` 生成一个渲染尺度：

- `domainMin/domainMax`：原始字段范围。
- `unitMin/unitMax`：渲染坐标范围，默认 `[-1, 1]`。
- `majorTicks`：nice 主刻度，约 5 到 8 个。
- `minorTicks`：主刻度之间的次刻度，不显示文字。
- `valueToUnit(value)`：将原始字段值映射到当前轴的 3D 坐标。

曲面高度轴因为 core 默认 `heightScale = 0.8`，由 Web demo 传入 `unitHalfRange = 0.8`，保证高度刻度和曲面几何对齐。

### AxisLayout3D

坐标轴采用稳定的前下左包围盒布局：

- X 轴：底面前边缘。
- Z depth 轴：底面侧边缘。
- Y height 轴：左前垂直边缘。
- 底面网格：随 X/Z 主刻度生成。
- 高度参考线：随 Y 主刻度生成。

后续可根据相机方向选择前景边，本次不做相机动态换边，避免引入交互抖动。

### 格式化

根据主刻度步长决定小数位：

- 大数使用 `k/M/B` 简写。
- 小数位由 step 决定，并去除末尾 0。
- 极小数使用科学计数法。

## 非目标

- 本次不实现 hover 坐标拾取。
- 本次不实现动态相机前景边选择。
- 本次不改变点云或曲面的 core 归一化输出。
